# 元老院群消息分类与执行规则

适用范围：
- channel = feishu
- chat_type = group
- chat_id = `oc_26487f6c792f445d4450570ddcef97a2`
- 且消息明确 @ 了 bot

## 分类顺序
1. 续办型
2. 正式事项型
3. 问答型

## 续办型
### 触发条件
- 显式包含 `JJC-...`
- 或出现：继续 / 接着 / 更新进度 / 汇报状态 / 基于上一条继续 / 拆成阶段
- 或引用/回复的是已存在任务消息

### 动作
- 不新建任务号
- 复用既有任务号
- 更新原 weiUI 条目
- 追加原流转

## 正式事项型
### 触发条件
- 显式说：正式事项 / 正式任务 / 写入看板 / 转中书省 / 纳入流程 / 立项 / 推进
- 或语义上明显需要多步骤推进、后续跟踪、正式产出

### 动作
- 新建任务号（JJC-...）
- 写入 weiUI
- 初始化流转
- 转 `taizi`

## 问答型
### 定义
- 既不是续办型，也不是正式事项型

### 动作
- 直接回答
- 不新建任务号
- 不写 weiUI
- 不启动正式流转

## 看板写入门槛
### 允许写 weiUI
- 正式事项型
- 续办型

### 禁止写 weiUI
- 问答型
- 自我介绍
- 架构说明
- 普通解释/讨论
- 闲聊

## 兜底原则
- 不确定时，先澄清，不要乱建任务
- 只有 `taizi` 超时/报错/空结果时，前台代理才自己兜底
